# glaza — видео → самодостаточный конспект (локально)

Claude смотрит обучающее видео **локально** и собирает **HTML-конспект**: текст-ядро из транскрипта, скриншоты — иллюстрации к уже сформулированным тезисам. Без облака — ничего не покидает машину.

> **Развилка ролей — прочти сначала:**
> - **Используешь `/glaza`** (смотришь видео → собираешь конспект)? → весь флоу в **`SKILL.md`**, инварианты в **`RULES.md`**. Карта репозитория ниже тебе не нужна.
> - **Дорабатываешь плагин** (правишь код/скилл/тесты)? → карта ниже для тебя.

## Структура

| Путь | Что это |
|---|---|
| `SKILL.md` | флоу использования — **источник истины** (Step 0…7 + стиль конспекта + стоимость) |
| `RULES.md` | 8 инвариантов флоу с обоснованиями — что нельзя нарушать и почему |
| `scripts/install.py` | установка под железо: детект → план → подтверждение → pip/модель/`.env` |
| `scripts/hardware.py` | детект ОС/GPU/VRAM/RAM (только факты, решения — в `install.py`) |
| `scripts/glaza.py` | оркестратор: скачать → `fps`-нарезка → pHash-дедуп → транскрипт → отчёт |
| `scripts/frames.py` | нарезка кадров: равномерная (`extract_fps1`), по сменам сцены, один кадр в родном разрешении (`extract_one`) |
| `scripts/dedup.py` | dHash + схлопывание серий; представитель серии = **последний** кадр |
| `scripts/dense_zones.py` | detect-зоны по журналу + плотная перенарезка, бюджет кадров |
| `scripts/transcribe.py` | диспетчер Whisper (`faster` / `ov` / `whispercpp`) |
| `scripts/inline_images.py` | `<img src>` → base64, самодостаточный HTML |
| `scripts/source.py`, `env.py`, `setup.py` | источник видео (yt-dlp/файл), `.env`, preflight |
| `tests/` | единственный набор тестов, `unittest` из stdlib |

## Тесты

```bash
python -m unittest discover -s tests -t .
```

Только stdlib — pytest не нужен. Тесты, которым нужен реальный `ffmpeg`/`Pillow`, сами пропускаются, если их нет.

## Настройка

Все настройки — в `.env` (создаётся `install.py`, шаблон — `.env.example`). Значимые ключи и их цена описаны в `README.md`. Личные пути живут только в `.env`, он в `.gitignore`.

## Куда добавлять свой Whisper-бэкенд

Бэкенд изолирован за одним интерфейсом `transcribe(audio, lang, backend, cfg)`. Добавь `_yourbackend.py`, возвращающий `[{start, end, text}]`, подключи его в диспетчер `transcribe.py` и добавь ветку `WATCH_WHISPER=yourbackend` в `setup.py` (preflight) и в `install.py` (матрица железа).
