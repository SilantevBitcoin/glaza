# /glaza — локальный content-aware просмотр видео для Claude

Claude смотрит обучающее видео **локально** и собирает **самодостаточный HTML-конспект**: текст — ядро (из локального Whisper-транскрипта), а скриншоты добавляются только там, где действительно нужно посмотреть на экран. Без облака — ничего не покидает вашу машину.

## Установка

```bash
git clone https://github.com/SilantevBitcoin/glaza
cd glaza
python scripts/install.py
```

`install.py` определяет ваше железо, подбирает Whisper-бэкенд и модель под объём памяти, **показывает план и спрашивает подтверждение**, затем ставит пакеты, скачивает модель и пишет `.env`. Посмотреть план, ничего не устанавливая: `python scripts/install.py --dry-run`.

Единственное, что он **не** ставит сам, — `ffmpeg` (нужен пакетный менеджер и права). Если его нет, скрипт назовёт одну команду под вашу ОС (`winget` / `brew` / `apt`) и остановится.

### Подключение к Claude Code

**Как плагин** — вызывается `/glaza:glaza`:
```
/plugin marketplace add SilantevBitcoin/glaza
```

## Зависимости

Всё, кроме `ffmpeg`, ставит `install.py`: `Pillow` (pHash-дедуп кадров), `yt-dlp` (только для ссылок) и пакеты выбранного Whisper-бэкенда.

`python scripts/setup.py --check` проверяет готовность **до закачки видео** и называет конкретную причину, если что-то не так.

## Настройки, которые важны (`.env`)
| ключ | по умолчанию | зачем |
|---|---|---|
| `WATCH_LANG` | `ru` | Транскрипция **на языке оригинала** (`task=transcribe`, без перевода). `ru` распознаёт русскую речь и оставляет английские названия инструментов как есть. Поставить `en` на русское аудио — Whisper начнёт подбирать английские слова, которые просто *похожи по звуку*, и смысл вывернется наизнанку. `auto` может ошибиться на интро или музыке и утащить весь транскрипт в чужой язык. |
| `WATCH_RES_OVERVIEW` | `1568` | Это **потолок ширины, а не цель**. Кадры не апскейлятся: 720p-видео останется 1280 px. Растянутый кадр стоит субагенту столько же токенов (`⌈w/28⌉×⌈h/28⌉`) и не добавляет ни одной детали. Финальные скрины от параметра не зависят — Step 6 переизвлекает их в родном разрешении. |
| `WATCH_FPS` | `1` | Плотность нарезки; повышать только для demo-зон. |
| `WATCH_DEDUP_THRESHOLD` | `10` | Расстояние pHash (0..64). Ниже — сохраняются мелкие, но значимые изменения (тумблер включился); `0` выключает дедуп. |

## Whisper-бэкенды

`install.py` выбирает бэкенд сам; таблица — на случай, если хотите поменять руками.

| бэкенд | железо | что ставится |
|---|---|---|
| `faster` | NVIDIA (CUDA) или CPU | `faster-whisper` + `nvidia-cublas-cu12`, `nvidia-cudnn-cu12` (для CUDA); модель скачивается при первом запуске |
| `ov` | Intel Arc / OpenVINO | `openvino-genai`, `librosa`; готовая модель `OpenVINO/whisper-large-v3-fp16-ov` (конвертировать ничего не нужно) |
| `whispercpp` | Apple Silicon (Metal), AMD (Vulkan) | бинарь `whisper-cli` (на macOS: `brew install whisper-cpp`) + `ggml`-модель |

### Модель под объём памяти

| Железо | Модель |
|---|---|
| NVIDIA, VRAM ≥ 8 ГБ | `large-v3` |
| NVIDIA, VRAM 4–8 ГБ | `medium` |
| NVIDIA, VRAM < 4 ГБ | `small` |
| Apple / AMD | по RAM: ≥16 ГБ → `large-v3`, ≥8 → `medium`, иначе `small` |
| Только CPU | `small` (при RAM < 8 ГБ — `base`) |

**`FW_DEVICE` не оставляйте в `auto`, если у вас NVIDIA.** Значение `auto` определяет наличие CUDA через `import torch`, а torch в зависимостях нет — без него транскрипция молча уедет на CPU. `install.py` пишет `FW_DEVICE=cuda` явно.

## Как это работает
1. `yt-dlp` скачивает видео **один раз** (или вы передаёте локальный файл).
2. `ffmpeg` режет кадры равномерно (`fps=1` — плотность привязана к длине видео, лимита кадров нет; апскейла нет), затем **pHash-дедуп** схлопывает почти одинаковые серии. Представитель серии — её **последний** кадр: завершённое состояние (тумблер включён, слайд дорисован).
3. Локальный Whisper-бэкенд расшифровывает аудио в сегменты с таймкодами, на языке оригинала. Если бэкенд упал, `glaza.py` говорит об этом прямо и выходит с ненулевым кодом — конспект без текста не конспект.
4. `glaza.py` печатает markdown-отчёт: уникальные кадры-представители (со временем их показа) и транскрипт.
5. Claude следует `SKILL.md`: раздаёт кадры пачками **субагентам**, те возвращают текстовый журнал состояний экрана (основной контекст не забивается картинками); пишет конспект из транскрипта + журнала; переизвлекает выбранные скриншоты в **родном разрешении**; собирает один самодостаточный `digest.html` (тёмная/светлая тема, кликабельные скрины). Скачанное видео удаляется в конце.

## Собрать под себя
Бэкенд транскрипции изолирован за единым интерфейсом (`transcribe(audio, lang, backend, cfg)`). Чтобы добавить свой: создайте `_yourbackend.py`, возвращающий `[{start,end,text}]`, подключите его в диспетчер `transcribe.py` и добавьте ветку `WATCH_WHISPER=yourbackend`. Личные/приватные пути — только в `.env` (в `.gitignore`).
