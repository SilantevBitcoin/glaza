# Поддержка публичной копии — как готовить к публикации

> **Кому.** Мейнтейнеру (владельцу или его ИИ-ассистенту), который обновляет ЭТУ публичную копию из рабочей (приватной). Пользователю репозитория это не нужно — ему `README.md`.

Эта папка — **публичная копия** glaza: код, скилл и тесты без личной памяти разработчика, лесов разработки и конкретики окружения. Рабочая копия живёт отдельно и приватно.

## Главное правило

**Эта копия не правится руками.** Она пересобирается из рабочей одной командой, и любая правка, сделанная здесь, будет затёрта следующей сборкой. Источник истины — рабочая копия; чинить надо там.

Сборщик (`tools/publish.py`) живёт **в рабочей копии** и сюда намеренно не переносится: постороннему он бесполезен (пересобирать не из чего), а внутри у него по необходимости лежат имя владельца и образцы личных путей — ровно то, что он обязан ловить.

```bash
# запускается ИЗ РАБОЧЕЙ копии
python tools/publish.py            # пересобрать публичную папку + прогнать гейт
python tools/publish.py --check    # только гейт, ничего не пересобирая
```

Почему автоматически, а не вручную: рабочая и публичная копии делят **код**. Ручное копирование `scripts/` при каждом багфиксе рано или поздно приводит к разъезду копий — а разъехавшийся код даёт баг, который воспроизводится только у пользователей.

## Что переносится

Копируется **только** то, что перечислено в `WHITELIST` внутри `tools/publish.py` (белый список, не чёрный: чёрный пропустил бы любой новый файл):

`SKILL.md`, `RULES.md`, `README.md`, `README.ru.md`, `LICENSE`, `NOTICE`, `.env.example`, `.gitignore`, `.claude-plugin/`, `scripts/`, `tests/`

Плюс три файла, которые существуют только здесь и берутся из шаблонов рабочей копии: `CLAUDE.md`, `MAINTAINING.md`, `CHANGELOG.md`.

**Не переносится:** память разработчика, спеки и планы, задания агентам, `.env`, сам сборщик и его тесты.

## Правила адаптации

| В рабочей копии | В публичной |
|---|---|
| Личный путь (`<диск>:\...`, домашний каталог) | относительный путь либо плейсхолдер + инструкция по установке |
| Личная конфигурация бэкенда | только `.env.example` с плейсхолдерами; настоящие значения ставит `install.py` |
| Имя владельца в коде/скилле | «владелец» (ФИО — только в `LICENSE`, `NOTICE`, `README`, манифестах плагина) |
| Память разработчика, спеки, задания агентам | не переносятся вовсе |

## Гейт (проверяется автоматически)

`tools/publish.py` откажется публиковать, если найдёт:

1. личный путь (диск + `AI`/`Users`, домашний каталог);
2. носитель секретов (`.env`, `*.key`, `*.pem`, `credentials*`, `*.secret`);
3. файл вне белого списка;
4. имя владельца вне `LICENSE` / `NOTICE` / `README` / манифестов;
5. красные тесты в самой копии.

Любое срабатывание — ненулевой код возврата и никакого коммита.

## Публикация

Личная почта не должна попадать в публичную историю — автор коммитов под **noreply**-адресом GitHub:

```bash
git config --local user.email "164205213+SilantevBitcoin@users.noreply.github.com"
git config --local user.name  "Egor Silantev"
git add -A
git commit -m "sync: <что перенесено>"
git push origin main
```

После пуша проверь, что личной почты в истории нет:

```bash
git log --format='%an <%ae>' | sort -u
```
